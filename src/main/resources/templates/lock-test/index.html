시나리오 2: ShedLock 배치 중복 실행 방지

여러 서버에서 동시에 배치가 실행될 때 단 하나만 실행된다.
CM_SHEDLOCK 테이블에 잠금 정보를 기록해 분산 환경에서 중복 방지.

동작 원리

  1. 서버 A: INSERT INTO CM_SHEDLOCK 성공 → 배치 실행
  2. 서버 B: INSERT 실패 (이미 잠김) → 배치 SKIP
  3. 배치 완료 후 → LOCK_UNTIL 갱신

스케줄러 설정

@Scheduled(fixedDelay = 30_000)
@SchedulerLock(
  name = "contentProcessingBatch",
  lockAtMostFor = "25s",
  lockAtLeastFor = "5s"
)
💡 서버 실행 로그에서 [BATCH] [LOCK ACQUIRED]를 확인하세요. 서버 2대를 기동하면 하나만 실행되는 것을 확인할 수 있습니다.
