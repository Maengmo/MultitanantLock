ShedLock 동작 원리

분산 락 흐름

  1. 배치 시작 → CM_SHEDLOCK에 lock_until 기록
  2. 다른 서버: 같은 배치 시도 → lock_until이 미래 → SKIP
  3. 배치 완료 → lock_until을 현재 시각으로 갱신 (락 해제)
  4. 서버 다운 → lockAtMostFor 만료 후 자동 해제

현재 설정

@Scheduled(fixedDelay = 30_000)
@SchedulerLock(
  name = "contentProcessingBatch",
  lockAtMostFor = "25s",   // 서버 다운 시 자동 해제
  lockAtLeastFor = "5s"    // 최소 락 유지
)
